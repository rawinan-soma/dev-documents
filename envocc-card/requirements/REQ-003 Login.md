# REQ-003 Login
### Use case
- เข้าสู่ระบบ
### ผู้ใช้งานหลัก
- user
- unit admin
- prov/dep admin
### Objectives
- เพื่อให้ผู้ใช้งานทั้งหมดเข้าสู่ระบบ ในการดำเนินการกับระบบต่อไป
### Functional Requirements
1. ระบบจะต้องมีช่องทางในการเข้าสู่ระบบ แยกกันทั้ง user และ admin
2. ระบบต้องมีการตรวจสอบว่ามี user นั้นอยู่ในระบบ
3. ระบบต้องมีการตรวจสอบความถูกต้องของ username และ password เมื่อมีการร้องขอเข้าสู่ระบบ
### Pre-conditions
- ก่อนที่จะมีการเข้าสู่ระบบ ระบบจะต้องมีข้อมูลของผู้ใช้งานอยู่ในฐานข้อมูล
- ในกรณีของ user ที่เป็นผู้ร้องขอมีบัตร ระบบจะต้องมีข้อมูลของ user นั้น และได้รับการยืนยันตัวตนจาก admin ก่อน
### Post-conditions
- ถ้ามีการเข้าใช้งานด้วย username และ password ที่ถูกต้อง ระบบต้องยอมให้ผู้ใช้เข้าสู่ระบบได้
### Alternate conditions
- ไม่มี
### Exception conditions
- ถ้ามีการเข้าใช้งานด้วย username หรือ password ที่ไม่ถูกต้อง ระบบต้องปฏิเสธและแจ้งเตือนผู้ใช้
