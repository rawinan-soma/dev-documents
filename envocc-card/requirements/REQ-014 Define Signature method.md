# REQ-013 Define signature method
### Use case
- เลือกวิธีการลงนาม
### ผู้ใช้งานหลัก
- prov/dep admin
### Objectives
- เพื่อเลือกวิธีการลงนามในบัตรพนักงานเจ้าหน้าที่
### Functional Requirements
1. ระบบจะต้องมีช่องทางในการเลือกวิธีการลงนาม
2. ระบบจะต้องแสดงตัวเลือกของวิธีการลงนาม
3. ระบบจะต้องให้ admin เลือกวิธีการลงนามได้เพียง 1 วิธี
4. ระบบจะต้องมีการตรวจสอบสถานะของคำร้องขอมีบัตร
### Pre-conditions
- ก่อนที่จะมีการเลือกวิธีลงนาม
	- admin จะต้องเข้าสู่ระบบ
	- สถานะของคำร้องจะต้องอยู่ในสถานะ เลือกวิธีลงนาม
### Post-conditions
- ถ้า admin
	- เข้าสู่ระบบ และ
	- เลือกวิธีการลงนาม
- ระบบจะต้อง
	- บันทึกข้อมูลลงในฐานข้อมูล
	- เปลี่ยนแปลงสถานะของคำร้องขอมีบัตร
### Alternate conditions
- ไม่มี
### Exception conditions
- ไม่มี
