# REQ-014 Document approval
### Use case
- ตรวจสอบเอกสาร
- สั่งแก้ไขเอกสาร
### ผู้ใช้งานหลัก
- prov/dep admin
### Objectives
- เพื่อตรวจสอบเอกสารที่ unit admin upload เข้ามา ก่อนที่จะดำเนินการขั้นต่อไป
### Functional Requirements
1. ระบบจะต้องมีช่องทางให้เข้าถึงการตรวจสอบเอกสารและสั่งแก้ไขเอกสาร
2. ระบบจะต้องแสดงรายการเอกสารที่ต้องตรวจสอบทั้งหมด
3. ระบบจะต้องแสดงไฟล์เอกสารที่ต้องการตรวจสอบ
4. ระบบจะต้องให้ admin สามารถเลือกอนุมัติเอกสารทั้งหมดหรือไม่
5. ระบบจะต้องให้ admin สั่งแก้ไขเอกสารได้
### Pre-conditions
- ก่อนที่จะตรวจสอบเอกสาร หรือ สั่งแก้ไขเอกสาร
	- prov/dep admin จะต้องเข้าสู่ระบบ
	- unit admin จะต้อง upload เอกสารแล้ว
	- สถานะคำร้องจะต้องอยู่ในสถานะ ตรวจสอบเอกสาร
### Post-conditions
- ถ้า prov/dev admin ตรวจสอบเอกสารและอนุมัติ
	- ระบบจะต้องบันทึกการเปลี่ยนแปลง
	- ระบบจะต้อง update สถานะคำร้องขอมีบัตรต่อไป
### Alternate conditions
- ถ้า prov/dep admin ตรวจสอบเอกสาร และไม่อนุมัติ
	- prov/dep admin จะสามารถสั่งแก้ไขเอกสารได้
	- ระบบจะต้อง update สถานะคำร้องขอมีบัตรต่อไป
	- ระบบจะต้องลบเอกสารทั้งหมดของคำร้องนั้น
	- unit admin จะต้องตรวจสอบเอกสารและ upload เข้ามาใหม่
### Exception conditions
- ไม่มี
