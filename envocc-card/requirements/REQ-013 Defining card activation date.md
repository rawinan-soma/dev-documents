# REQ-012 Defining card activation date
### Use case
- เลือกวันที่อนุมัติ
### ผู้ใช้งานหลัก
- prov/dep admin
### Objectives
- เพื่อเลือกวันที่บัตรพนักงานเจ้าหน้าที่จะมีผลบังคับใช้
### Functional Requirements
1. ระบบจะต้องมีช่องทางในการตั้งวันที่อนุมัติ
2. ระบบจะต้องแสดง UI การเลือกวันที่อนุมัติ
3. ระบบจะต้องตรวจสอบวันที่อนุมัติก่อนบันทึกลงฐานขอมูล
4. ระบบจะต้องมีการตรวจสอบสถานะของคำร้องขอมีบัตร
5. ระบบจะต้องคำนวนวันหมดอายุบัตรหลังจากการตั้งวันที่อนุมัติ
### Pre-conditions
- ก่อนที่จะตั้งวันที่อนุมัติ
	- admin ต้องเข้าสู่ระบบ และ
	- สถานะของคำร้องจะต้องอยู่ในสถานะ ตั้งวันที่อนุมัติ
### Post-conditions
- ถ้า admin 
	- เข้าสู่ระบบ และ
	- ตั้งวันที่อนุมัติถูกต้อง
- ระบบจะต้อง
	- คำนวนวันหมดอายุ
	- บันทึกวันที่อนุมัติและวันหมดอายุลงในฐานข้อมูล
### Alternate conditions
- ไม่มี
### Exception conditions
- ถ้า admin ตั้งวันที่อนุมัติไม่ถูกต้อง ระบบจะต้องปฏิเสธการตั้งวันที่อนุมัติ และแจ้งเตือนไปยัง admin
