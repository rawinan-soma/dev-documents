# REQ-001 User registration
### Use case
- ลงทะเบียน
### ผู้ใช้งานหลัก
- user ผู้ร้องขอมีบัตร
### Objectives
- เพื่อบันทึกข้อมูลของผู้ร้องขอมีบัตรในระบบ
### Functional Requirements
1. ระบบต้องมีช่องทางให้ user เข้ามาลงทะเบียน
2. ระบบต้องให้ user กรอกข้อมูลที่จำเป็นทั้งหมด
3. ระบบต้องตรวจสอบข้อมูลที่จำเป็นให้ครบถ้วนก่อนลงทะเบียน
4. ระบบต้องมีการตรวจสอบข้อมูลให้ถูกต้องตามรูปแบบทั้งหมดก่อนลงทะเบียน
5. ระบบต้องมีการตรวจสอบการลงทะเบียนซ้ำในฐานข้อมูล
### Pre-conditions
- ก่อนที่จะลงทะเบียน จะต้องไม่มีข้อมูลของ user ในระบบ
### Post-conditions
- ถ้า user กรอกข้อมูลที่จำเป็นครบถ้วน และถูกต้องตามรูปแบบ ระบบจะต้องบันทึกข้อมูลลงในฐานข้อมูล
### Alternate conditions
- ไม่มี
### Exception conditions
- ถ้า user กรอกข้อมูลซ้ำกับที่มีในฐานข้อมูล ระบบจะปฏิเสธการลงทะเบียนและแจ้งเตือน user
- ถ้า user กรอกข้อมูลที่จำเป็นไม่ครบถ้วน ระบบจะต้องปฏิเสธการลงทะเบียน และแจ้งเตือน user
- ถ้า user กรอกข้อมูลไม่ถูกต้องตามรูปแบบ ระบบจะต้องปฏิเสธการลงทะเบียน และแจ้งเตือน user
