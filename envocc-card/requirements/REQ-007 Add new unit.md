# REQ-007 Add new unit
### Use case
- เพิ่มหน่วยบริการ
### ผู้ใช้งานหลัก
- Prov/Dep admin
### Objectives
- เพื่อเพิ่มข้อมูลของหน่วยบริการ ที่จะเป็นต้นสังกัดของพนักงานเจ้าหน้าที่
### Functional Requirements
1. ระบบจะต้องมีช่องทางให้ admin เพิ่มหน่วยบริการได้
2. ระบบจะต้องมีการกรอกข้อมูลในการเพิ่มหน่วยบริการ
3. ระบบจะต้องมีการตรวจสอบข้อมูลตามรูปแบบที่กำหนดไว้ก่อนเพิ่มหน่วยบริการ
4. ระบบจะต้องมีการตรวจสอบข้อมูลของหน่วยบริการก่อนเพิ่มหน่วยบริการ
### Pre-conditions
- ก่อนที่จะมีการเพิ่มหน่วยบริการ admin 
	- admin จะต้องทำการเข้าสู่ระบบ
	- จะต้องไม่มีข้อมูลของหน่วยบริการนั้นๆ ในระบบ
### Post-conditions
- ถ้า admin กรอกข้อมูลได้ครบถ้วนและถูกต้อง ระบบจะต้องเพิ่มข้อมูลของหน่วยบริการลงในฐานข้อมูล
### Alternate conditions
- ไม่มี
### Exception conditions
- ถ้า admin กรอกข้อมูลผิด หรือไม่ครบถ้วน
	- ระบบจะต้องปฏิเสธการเพิ่มหน่วยบริการ
	- ระบบจะต้องแจ้งเตือนไปยัง admin
- ถ้า admin เพิ่มหน่วยบริการที่มีข้อมูลอยู่แล้วในระบบ
	- ระบบจะต้องปฏิเสธการเพิ่มหน่วยบริการ
	- ระบบจะต้องแจ้งเตือนไปยัง admin
