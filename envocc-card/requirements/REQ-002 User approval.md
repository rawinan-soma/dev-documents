# REQ-002 User approval
### Use case
- อนุมัติการลงทะเบียน
### ผู้ใช้งานหลัก
- Unit admin ผู้ดูแลหน่วยบริการ
### Objectives
- เพื่อยืนยันตัวตนของผู้ร้องขอมีบัตร
- เพื่อให้ผู้ร้องขอมีบัตรดำเนินการต่อได้
### Functional Requirements
1. ระบบต้องให้ unit admin ดูรายชื่อและข้อมูลเบื้องต้นของ user ที่รอการยืนยันได้
2. ระบบต้องให้ unit admin สามารถยืนยันตัวตนของ user ได้
3. ระบบต้องให้ unit admin สามารถปฏิเสธการยืนยันตัวตนของ user ได้
### Pre-conditions
- ก่อนที่จะมีการยืนยันตัวตน จะต้องมีข้อมูลของ user อยู่ในระบบแล้ว
### Post-conditions
- ถ้า admin ยืนยันตัวตนของ user แล้ว ระบบจะต้องบันทึกข้อมูลและสถานะลงในฐานข้อมูล
### Alternate conditions
- ถ้า admin ปฏิเสธการยืนยันตัวตนของ user ระบบจะต้องลบข้อมูลของ user นั้นออกจากระบบ
### Exception conditions
- ไม่มี
