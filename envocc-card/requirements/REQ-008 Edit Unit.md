# REQ-008 Edit unit
### Use case
- แก้ไขข้อมูลหน่วยบริการ
### ผู้ใช้งานหลัก
- Prov/Dev admin
### Objectives
- เพื่อแก้ไขข้อมูลของหน่วยบริการให้ถูกต้อง
### Functional Requirements
1. ระบบจะต้องมีช่องทางการเข้าถึงการแก้ไขข้อมูลหน่วยบริการ
2. ระบบจะต้องแสดงข้อมูลหน่วยบริการก่อนที่จะถูกแก้ไข
3. ระบบจะต้องมีการตรวจสอบการกรอกข้อมูลให้ถูกต้องตามที่กำหนด
### Pre-conditions
- ก่อนที่จะแก้ไขข้อมูลหน่วยบริการ 
	- admin จะต้องเข้าสู่ระบบ
	- ระบบจะต้องมีข้อมูลหน่วยบริการนั้นในฐานข้อมูล
### Post-conditions
- ถ้า admin กรอกข้อมูลที่จำเป็นครบถ้วนและถูกต้องตามรูปแบบที่กำหนดไว้ ระบบจะต้องมีการบันทึกการเปลี่ยนแปลงไว้ในฐานข้อมูล
### Alternate conditions
- ถ้า admin ยกเลิกการเปลี่ยนแปลง ระบบจะต้องไม่มีการบันทึกการเปลี่ยนแปลงใดๆ ลงในฐานข้อมูล
### Exception conditions
- ถ้า admin กรอกข้อมูลที่จำเป็นไม่ครบถ้วน หรือ ไม่ถูกต้องตามรูปแบบที่กำหนดไว้
	- ระบบจะต้องปฏิเสธการแก้ไขข้อมูล
	- ระบบจะต้องแจ้งเตือนไปยัง admin
