# REQ-006 Add new admin
### Use case
- เพิ่ม admin ระดับรอง
### ผู้ใช้งานหลัก
- Prov/Dep admin
### Objectives
- เพื่อเพิ่ม admin ในระดับที่รองจากตัว admin เดิม เช่น admin สสจ เพิ่ม admin ระดับหน่วยบริการ
### Functional Requirements
1. ระบบจะต้องมีช่องทางให้ admin เพิ่ม admin ระดับรองลงมา
2. ระบบจะต้องให้ admin กรอกข้อมูลที่จำเป็นทั้งหมด
3. ระบบจะต้องมีการตรวจสอบข้อมูลที่จำเป็นให้ครบถ้วน
4. ระบบจะต้องมีการตรวจสอบรูปแบบของข้อมูล
5. ระบบจะต้องมีการยืนยันการเข้าสู่ระบบของ admin ก่อนที่จะมีการเพิ่ม admin คนใหม่
6. ระบบจะต้องมีการตรวจสอบการลงทะเบียน admin ซ้ำ
### Pre-conditions
- ก่อนที่จะมีการลงทะเบียน admin คนใหม่ จะต้องมีการเข้าสู่ระบบของ admin และในระบบจะต้องมีข้อมูลของหน่วยงานที่จะเพิ่ม admin คนใหม่
### Post-conditions
- ถ้า admin กรอกข้อมูลที่จำเป็นครบถ้วนและกรอกข้อมูลถูกต้องตามรูปแบบ ระบบจะต้องบันทึกข้อมูล admin คนใหม่ลงในฐานข้อมูล
- ถ้ามีการบันทึกข้อมูลของ admin คนใหม่ลงในฐานข้อมูลแล้ว ระบบจะต้องยอมให้ admin คนใหม่เข้าสู่ระบบได้
### Alternate conditions
- ไม่มี
### Exception conditions
- ถ้า admin กรอกข้อมูลที่จำเป็นไม่ครบถ้วน หรือกรอกข้อมูลไม่เป็นไปตามรูปแบบที่กำหนดไว้ ระบบจะต้องปฏิเสธการเพิ่ม admin คนใหม่ และมีการแจ้งเตือน
