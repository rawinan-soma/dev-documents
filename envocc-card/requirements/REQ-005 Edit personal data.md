# REQ-005 Edit personal data
### Use case
- แก้ไขข้อมูลส่วนตัว
- แก้ไขเอกสาร
### ผู้ใช้งานหลัก
- user
- admin
### Objectives
- เพื่อให้ผู้ใช้งานแก้ไขข้อมูลส่วนตัวให้ถูกต้องได้
- เพื่อให้ user ที่ร้องขอมีบัตร มีข้อมูลที่ถูกต้อง
### Functional Requirements
1. ระบบจะต้องสามารถเข้าถึงการแก้ไขข้อมูลส่วนตัวได้
2. ระบบจะต้องแสดงข้อมูลของผู้ใช้งานที่มีอยู่เดิมได้
3. ระบบจะต้องตรวจสอบการมีอยู่ข้อมูลที่จำเป็นหลังจากการแก้ไข
4. ระบบจะต้องตรวจสอบรูปแบบข้อมูลหลังจากการแก้ไข
### Pre-conditions
- ก่อนที่จะทำการแก้ไขข้อมูล ระบบจะต้องมีข้อมูลของผู้ใช้งาน และ ผู้ใช้งานต้องเข้าสู่ระบบแล้ว
### Post-conditions
- ถ้าผู้ใช้งานแก้ไขข้อมูลโดยที่มีข้อมูลในช่องที่จำเป็น และถูกต้องตามรูปแบบแล้ว ระบบจะต้องบันทึกการเปลี่ยนแปลงนั้นลงในฐานข้อมูล
### Alternate conditions
- ไม่มี
### Exception conditions
- ถ้า user แก้ไขข้อมูลโดยไม่กรอกข้อมูลที่จำเป็น หรือ กรอกข้อมูลไม่ถูกต้องตามรูปแบบ ระบบจะต้องปฏิเสธการแก้ไขข้อมูลครั้งนั้น และแจ้งเตือนไปยัง user
