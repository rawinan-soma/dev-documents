# REQ-010 Document printing
### Use case
- พิมพ์เอกสารประสบการณ์
- พิมพ์เอกสารคำร้องขอมีบัตร
- พิมพ์บัตรพนักงานเจ้าหน้าที่ที่อนุมัติแล้ว
### ผู้ใช้งานหลัก
- prov/dep admin
- user ผู้ร้องขอมีบัตร
### Objectives
- เพื่อพิมพ์เอกสารไปดำเนินการต่อ
- เพื่อพิมพ์บัตรที่อนุมัติแล้วส่งให้ user ผู้ร้องขอมีบัตร
### Functional Requirements
1. ระบบจะต้องมีช่องทางให้ผู้ใช้เข้าถึงการพิมพ์บัตร
2. ระบบจะต้องตรวจสอบข้อมูลที่จะนำมาใส่เอกสารก่อนพิมพ์
3. ระบบจะต้องตรวจสอบสถานะขั้นตอนของการร้องขอ
4. ระบบจะต้องแสดงภาพตัวอย่างก่อนพิมพ์เอกสาร
### Pre-conditions
- ก่อนที่จะทำการพิมพ์เอกสารได้
	- admin หรือ user ที่ร้องขอการพิมพ์เอกสารต้องเข้าสู่ระบบ และ
	- ต้องมีข้อมูลของเอกสารที่ต้องการพิมพ์อยู่ในระบบ
### Post-conditions
- ถ้า admin หรือ user เข้าสู่ระบบและร้องขอการพิมพ์เอกสาร ระบบจะต้องทำการพิมพ์เอกสารออกมาได้
### Alternate conditions
- admin หรือ user สามารถร้องขอการพิมพ์เอกสารเป็น ไฟล์ .PDF ได้
### Exception conditions
- ไม่มี
