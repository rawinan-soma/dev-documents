# REQ-002 User approval
### Use case
- อนุมัติการลงทะเบียน
### ผู้ใช้งานหลัก
- admin DOED
### Objectives
- อนุมัติการลงทะเบียนของสปก.
- เพื่อให้สปก. เข้าสู่ระบบต่อไปได้
### Functional Requirements
- ระบบจะต้องให้ admin ดูรายชื่อของสปก. ที่ลงทะเบียนเข้ามาทั้งหมด
- ระบบจะต้องให้ admin ดูรายละเอียดของสปก. ที่ลงทะเบียนเข้าก่อนที่จะอนุมัติการลงทะเบียน
- ระบบจะต้องให้ admin เลือกได้ว่าจะอนุมัติการลงทะเบียนของสปก.นั้นๆ หรือไม่
### Pre-conditions
- ก่อนที่จะมีการอนุมัติการลงทะเบียน
  - จะต้องมีการลงทะเบียนของสปก.ในระบบ และ
  - admin จะต้องเข้าสู่ระบบ
### Post-conditions
- ถ้า admin ยืนยันการลงทะเบียนแล้ว
  - ระบบจะต้องให้ user สปก.เข้าสู่ระบบได้
- ถ้า admin ปฏิเสธการลงทะเบียน
  - ระบบจะต้องลบข้อมูลการลงทะเบียนของสปก.นั้นออกจากฐานข้อมูล
### Alternate conditions
- ไม่มี
### Exception conditions
- ไม่มี
