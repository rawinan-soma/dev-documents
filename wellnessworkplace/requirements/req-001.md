# REQ-001 user registration
### Use case
- ลงทะเบียน สปก. 
### ผู้ใช้งานหลัก
- เจ้าหน้าที่ในสถานประกอบการ
### Objectives
- ให้สถานประกอบการมีข้อมูลอยู่ในระบบฯ เพื่อที่จะยืนยันการเข้าร่วมโครงการต่อไป
### Functional Requirements
1. ระบบจะต้องมีช่องทางให้ผู้ใช้งานเข้าถึงการลงทะเบียน
2. ระบบจะต้องให้ผู้ใช้งานกรอกข้อมูลที่จำเป็นทั้งหมด
3. ระบบจะต้องมีการตรวจสอบข้อมูลตามรูปแบบ
4. ระบบจะต้องมีการตรวจสอบการลงทะเบียนซ้ำ
### Pre-conditions
- ก่อนที่จะมีการลงทะเบียน ระบบจะต้องยังไม่มีข้อมูลของ สปก. อยู่ในระบบ
### Post-conditions
- ถ้า user 
  - กรอกข้อมูลถูกต้อง และ
  - ยืนยันการลงทะเบียน 
- ระบบจะต้องบันทึกข้อมูลไปยังฐานข้อมูล
### Alternate conditions
- ไม่มี
### Exception conditions
- ถ้า user 
  - ลงทะเบียนด้วยข้อมูลสปก. ซ้ำกับในฐานข้อมูล หรือ 
  - ลงทะเบียนด้วยข้อมูลที่ไม่ถูกต้องตามรูปแบบ 
- ระบบจะต้อง
  - ปฏิเสธการลงทะเบียน และ
  - แจ้งเตือนไปยัง user
